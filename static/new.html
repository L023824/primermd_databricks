KPIs in scope
One block per metric — name, definition, grain, time window, caveats, and where the SQL logic comes from. All of it flows into kpi_definitions.md and EXAMPLES.md.
SQL logic source — three modes
  • Use from playbook — points at one of the 12 Databricks-dialect SQL patterns; placeholders get resolved against the real schema in a live Claude.ai/Databricks MCP session. Every pattern a KPI points to gets its own section in EXAMPLES.md automatically — no separate pattern-picker step needed.
  • Paste existing SQL — wraps what you paste as the EXAMPLES.md stub, translated to Databricks dialect if needed. Since the Redshift → Databricks migration is still underway, most pasted SQL will still be Redshift dialect — check the "This is Redshift SQL" box to get a table mapping preview.
  • Describe rules — columns, business rules, exclusions/edge cases in plain English. Best when there's no existing SQL and live schema access isn't available yet.
Redshift → Databricks table mapping
  • Checking "This is Redshift SQL" scans your pasted SQL for table references and lets you confirm the Databricks catalog.schema.table each one maps to.
  • If a mapped table isn't already in your Databricks Target table list (previous tab), you'll get a prompt to add it there — or you can just correct the mapping if it's wrong.
